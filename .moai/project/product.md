# CypherLite - 제품 개요

> **SQLite의 단순함으로, Neo4j의 그래프 파워를**

---

## 프로젝트 개요

**CypherLite**는 경량 임베디드 단일 파일 그래프 데이터베이스 라이브러리입니다. SQLite가 관계형 데이터베이스를 민주화한 것처럼, CypherLite는 그래프 데이터베이스를 누구나 쉽게 사용할 수 있도록 설계되었습니다.

- **현재 상태**: 활성 개발 단계 (v1.0.0)
- **주요 언어**: Rust 1.92+ (2024 edition)
- **파일 확장자**: `.cyl` (CypherLite 고유 형식)

---

## 문제 정의

### 기존 그래프 데이터베이스의 한계

현재 그래프 데이터베이스 생태계는 두 가지 극단 사이에서 분열되어 있습니다.

**서버 기반 솔루션의 문제** (Neo4j, ArangoDB 등):
- 500MB 이상의 무거운 설치 용량
- 별도 서버 프로세스 필요 (운영 복잡도 증가)
- 설정 및 관리에 상당한 전문 지식 요구
- 소규모 애플리케이션이나 개발 환경에 과도한 오버헤드

**SQLite 기반 우회의 문제**:
- 그래프 쿼리(Cypher)를 SQL로 변환하는 복잡한 조인 쿼리 필요
- 인접 노드 탐색이 O(log n) 이상의 비용 발생
- 자연스러운 그래프 의미론 표현 불가

**경쟁 임베디드 솔루션의 공백**:
- KùzuDB: 2025년 10월 공식 아카이브 처리 (더 이상 활성 개발 없음)
- 시장에 플러그인 확장성을 갖춘 경량 임베디드 그래프 DB 부재

### CypherLite가 필요한 이유

그래프 데이터를 다뤄야 하는 개발자가 "단 한 줄의 함수 호출"로 데이터베이스를 시작할 수 있어야 합니다. 서버 설정, 포트 관리, 별도 프로세스 없이—단지 `.cyl` 파일 하나면 충분합니다.

## 핵심 설계 원칙

| 원칙 | 설명 |
|------|------|
| ZERO-CONFIG | 단일 함수 호출로 데이터베이스 오픈 |
| SINGLE-FILE | 전체 데이터베이스가 `.cyl` 파일 하나에 저장 |
| ACID COMPLIANT | WAL 기반 완전한 ACID 보장 |
| EMBEDDED | 인-프로세스 라이브러리, 별도 서버 불필요 |
| EXTENSIBLE | 플러그인 아키텍처로 무한 확장 |
| TEMPORAL-NATIVE | 시간 인식 쿼리 기본 지원 |
| CYPHER-STANDARD | openCypher 서브셋 지원 |

## 대상 사용자 및 활용 사례

### 1순위: LLM 에이전트 메모리 시스템

**페인 포인트**: LLM 에이전트는 의사결정, 상호작용 이력, 지식을 쿼리 가능한 형태로 저장해야 하지만, 기존 벡터 DB는 관계 탐색이 약하고 그래프 DB는 설치가 무겁습니다.

**CypherLite 솔루션**:
- 에이전트가 생성한 지식 그래프를 단일 파일에 영속화
- Cypher 쿼리로 관련 기억과 컨텍스트를 즉시 탐색
- 에이전트 프로세스에 임베드되어 네트워크 오버헤드 없음

### 2순위: 지식 그래프 애플리케이션 (GraphRAG)

**페인 포인트**: GraphRAG 파이프라인에서 도메인 지식의 의미론적 관계를 표현하고 탐색하기 위한 경량 솔루션 부재.

**CypherLite 솔루션**:
- 도메인 지식을 노드/엣지로 모델링
- Cypher로 다중 홉 관계 탐색
- 시간 인식 쿼리로 지식의 유효 시간 관리

### 3순위: 엣지 컴퓨팅 및 IoT

**페인 포인트**: 제한된 리소스 환경에서 센서 데이터 간의 관계 컨텍스트를 로컬로 처리해야 하는 요구.

**CypherLite 솔루션**:
- 소형 임베디드 시스템에서 동작 가능한 경량 설계
- 오프라인 동작 (네트워크 불필요)
- 단일 파일로 백업 및 이동 용이

### 4순위: 애플리케이션 임베디드 그래프

**페인 포인트**: 모바일 앱이나 데스크톱 앱에서 소셜 네트워크, 추천 시스템 같은 그래프 데이터를 로컬에서 처리해야 하는 요구.

**CypherLite 솔루션**:
- 앱 번들에 포함 가능한 단일 라이브러리
- Python/Node.js/C FFI 바인딩으로 다양한 플랫폼 지원

### 5순위: 테스트 및 개발 환경

**페인 포인트**: 그래프 DB가 필요한 기능을 개발/테스트할 때 매번 서버를 구동해야 하는 번거로움.

**CypherLite 솔루션**:
- 인프라 없이 즉시 그래프 프로토타이핑
- `tempfile` 기반 임시 DB로 격리된 테스트 환경 구성

## 핵심 기능 및 가치 제안

### 기능 요약

**스토리지 엔진**
- 4KB 페이지 기반 파일 형식 (SQLite 증명된 방식)
- 인덱스 프리 인접성 (Index-free Adjacency): O(1) 노드 탐색
- WAL(Write-Ahead Logging) 기반 충돌 복구
- MVCC 트랜잭션으로 다중 동시 읽기 지원
- 서브그래프 스토어: 엔티티 그룹핑 및 중첩 그래프 지원
- 하이퍼엣지 스토어: N:M 관계 지원 (노드, 서브그래프, 시간 참조 연결)

**쿼리 엔진**
- openCypher 서브셋 (v1.0: 80% 활용 사례 커버)
- 인라인 프로퍼티 필터: MATCH 패턴의 `{key: value}` 구문이 노드/관계/대상 노드 모두에서 정확하게 필터링됨
- 비용 기반 쿼리 최적화
- 결과 스트리밍 지원

**CI/CD 파이프라인**
- GitHub Actions 자동 품질 게이트 (test, clippy, fmt, MSRV, coverage, security)
- 커버리지 85% 미만 PR 자동 차단
- Dependabot 의존성 자동 업데이트

**시간 차원 (Temporal)**
- `AT TIME` 쿼리로 과거 상태 조회
- 버전 스토어로 노드/엣지 이력 관리
- 스냅샷 격리
- 시간 참조 (TemporalRef): `Node AT TIME timestamp`으로 과거 시점 노드 참조
- 하이퍼엣지 내 시간 참조로 시간 차원 그래프 관계 표현

**플러그인 시스템**
- 4가지 플러그인 타입: ScalarFunction, IndexPlugin, Serializer, Trigger
- 공통 `Plugin` 베이스 트레이트 + 제네릭 `PluginRegistry<T>`
- `cypherlite-core` 크레이트 내 `plugin/` 모듈로 통합 제공

**FFI 바인딩**
- C 헤더 자동 생성 (cbindgen)
- Python 바인딩 (PyO3)
- Node.js 바인딩 (neon)

### 가치 제안

1. **제로 설정**: `CypherLite::open("app.cyl")?` 한 줄로 시작
2. **이식성**: 단일 `.cyl` 파일 복사로 완전한 데이터베이스 이전
3. **성능**: Rust 구현으로 메모리 안전성과 고성능 동시 달성
4. **확장성**: 핵심 엔진을 건드리지 않고 플러그인으로 기능 추가

## 경쟁 포지셔닝

| 솔루션 | CypherLite 대비 차이점 |
|--------|----------------------|
| **Neo4j** | 500MB+ 서버 필요 vs. 경량 임베디드 (<50MB) |
| **KùzuDB** | 2025년 10월 아카이브 처리 vs. 활성 개발 + 플러그인 확장성 |
| **DuckDB+PGQ** | SQL 기반 그래프 확장 vs. 그래프 네이티브 (Cypher) |
| **SQLite** | 그래프 쿼리를 위한 복잡한 조인 vs. 인덱스 프리 인접성으로 O(1) 탐색 |

**CypherLite의 고유 포지션**: 경량 임베디드 + 그래프 네이티브 + Cypher 표준 + 플러그인 확장성

## 성공 지표

### 성능 목표 (v1.0)

| 지표 | 목표값 |
|------|--------|
| 단순 매치 쿼리 (p99) | < 10ms |
| 2홉 패턴 쿼리 (p99) | < 50ms |
| 바이너리 크기 | < 50MB |
| 메모리 (100만 노드) | < 500MB |
| 순차 쓰기 속도 | > 1,000 노드/초 |
| 동시 읽기 (4스레드) | > 50,000 읽기/초 |

### 제품 성공 지표

- **채택**: LLM 에이전트 프레임워크 3개 이상에서 CypherLite 통합
- **안정성**: v1.0 릴리즈 후 6개월간 크리티컬 버그 0건
- **생태계**: 커뮤니티 기여 플러그인 5개 이상
- **문서화**: 신규 개발자가 30분 내 첫 그래프 쿼리 실행 가능
