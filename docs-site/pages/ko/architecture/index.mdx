# 아키텍처

CypherLite는 네 개의 Rust crate로 구성된 계층형 아키텍처로 조직되어 있으며, 각 crate는 별도의 관심사를 담당합니다.

## 5계층 아키텍처

```
+--------------------------------------------------+
|              애플리케이션 계층                       |
|   Rust API  |  Python  |  Go  |  Node.js  |  C   |
+--------------------------------------------------+
|              FFI 계층 (cypherlite-ffi)              |
|   C ABI  |  PyO3  |  CGo  |  napi-rs              |
+--------------------------------------------------+
|           쿼리 엔진 (cypherlite-query)              |
|   Parser  |  Analyzer  |  Optimizer  |  Executor   |
+--------------------------------------------------+
|            코어 계층 (cypherlite-core)               |
|   그래프 모델  |  시간 여행  |  플러그인 시스템        |
+--------------------------------------------------+
|         스토리지 엔진 (cypherlite-storage)           |
|   B+Tree  |  WAL  |  BufferPool  |  PageManager    |
+--------------------------------------------------+
```

## Crate 의존성 그래프

```
cypherlite-ffi
    |
    v
cypherlite-query
    |
    v
cypherlite-core
    |
    v
cypherlite-storage
```

각 crate는 바로 아래 계층에만 의존하여 깔끔한 관심사 분리를 보장합니다:

| Crate | 책임 |
|-------|---------------|
| **cypherlite-storage** | 디스크 I/O, 페이지 관리, B+Tree 인덱스, WAL, 버퍼 풀, 크래시 복구 |
| **cypherlite-core** | 그래프 데이터 모델, 프로퍼티 타입, 시간 여행 버전 관리, 서브그래프, 하이퍼엣지, 플러그인 trait |
| **cypherlite-query** | Cypher 파서, 시맨틱 분석, 비용 기반 옵티마이저, volcano 실행기 |
| **cypherlite-ffi** | C ABI 익스포트, PyO3 바인딩, CGo 브릿지, napi-rs 애드온 |

## 스토리지 엔진

스토리지 엔진은 모든 그래프 데이터의 영속적 저장을 담당합니다.

### B+Tree 인덱스

- 노드와 엣지의 O(log n) 조회
- 관계 탐색을 위한 인덱스 프리 인접성
- 순서가 있는 프로퍼티 접근을 위한 범위 스캔 지원

### Write-Ahead Log (WAL)

- 모든 변경 사항은 데이터 페이지에 적용되기 전에 먼저 WAL에 기록
- 시작 시 크래시 복구가 커밋되지 않은 WAL 프레임을 재생
- 단일 작성자 / 다중 판독자 동시성 모델

### 버퍼 풀

- 쿼리 엔진과 디스크 사이에 위치한 LRU 페이지 캐시
- 더티 페이지 추적과 배치 쓰기 백
- 메모리 대 I/O 트레이드오프를 위한 설정 가능한 캐시 크기

### 페이지 매니저

- 고정 크기 페이지 할당 및 해제
- 공간 재사용을 위한 프리 리스트 관리
- `parking_lot` 기반 페이지 수준 잠금

## 쿼리 실행 파이프라인

하나의 Cypher 쿼리는 결과를 생성하기 전에 네 단계를 거칩니다:

```
Cypher 문자열
    |
    v
+-------------------+
|   1. 파서         |  재귀 하강 파싱, Pratt 표현식 파싱
+-------------------+  출력: 추상 구문 트리 (AST)
    |
    v
+-------------------+
|   2. 분석기       |  변수 스코프 검증, 레이블/타입 해석
+-------------------+  출력: 시맨틱 정보가 포함된 검증된 AST
    |
    v
+-------------------+
|   3. 옵티마이저   |  논리-물리 계획 변환, 프레디케이트 푸시다운
+-------------------+  출력: 물리 실행 계획
    |
    v
+-------------------+
|   4. 실행기       |  Volcano 이터레이터 모델, 12개 연산자
+-------------------+  출력: 결과 행 (스트리밍)
```

### 파서

- 수작업 재귀 하강 파서 (파서 생성기 미사용)
- 표현식 우선순위를 위한 Pratt 파싱
- 28개 이상의 Cypher 키워드 지원
- 인라인 프로퍼티 필터: `MATCH (n:Label {key: value})`

### 시맨틱 분석기

- `WITH` 경계를 넘는 변수 스코프 검증
- 레이블 및 관계 타입 해석
- 프로퍼티 비교를 위한 타입 체크

### 비용 기반 옵티마이저

- 논리-물리 계획 변환
- 중간 튜플 감소를 위한 프레디케이트 푸시다운
- 사용 가능한 B+Tree 인덱스에 기반한 인덱스 선택
- 다중 패턴 MATCH를 위한 조인 순서 최적화

### Volcano 실행기

- 이터레이터 기반 풀 모델 (지연 평가)
- 12개 물리 연산자: Scan, IndexScan, Filter, Project, Sort, Limit, Join, Create, Delete, Set, Merge, Aggregate
- openCypher 스펙에 따른 3값 논리 NULL 전파

## 동시성 모델

CypherLite는 SQLite에서 영감을 받은 **단일 작성자 / 다중 판독자** 모델을 사용합니다:

- 동시에 하나의 쓰기 트랜잭션만 허용 (뮤텍스를 통한 직렬화)
- 여러 동시 읽기 트랜잭션
- WAL 기반 스냅샷 격리로 판독자가 일관된 뷰를 볼 수 있도록 보장
- 쓰기 중 읽기 차단 없음 (판독자는 WAL 프레임 인덱스 사용)

<Callout type="info">
이 동시성 모델은 단일 프로세스에서 데이터베이스에 접근하는 임베디드 사용 사례에 이상적입니다. 다중 프로세스 접근의 경우 파일 잠금(OS 수준)을 사용하세요.
</Callout>

## 기능 플래그

CypherLite는 Cargo 기능 플래그를 사용하여 컴파일 시 선택적 기능을 활성화합니다:

| 플래그 | 기본값 | Crate | 설명 |
|------|---------|-------|-------------|
| `temporal-core` | 예 | core | 시간 여행 그래프 지원 |
| `temporal-edge` | 아니오 | core | 시간 여행 엣지 속성 |
| `subgraph` | 아니오 | core | 네임드 서브그래프 엔티티 |
| `hypergraph` | 아니오 | core | N:M 하이퍼엣지 관계 (`subgraph` 필요) |
| `full-temporal` | 아니오 | core | 모든 시간 여행 기능 |
| `plugin` | 아니오 | core | 플러그인 시스템 (ScalarFunction, IndexPlugin, Serializer, Trigger) |
