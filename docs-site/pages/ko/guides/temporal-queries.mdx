# 시간 여행 쿼리

CypherLite는 노드, 엣지, 프로퍼티의 모든 변경을 시간에 따라 추적합니다. 시간 여행 쿼리 구문을 사용하여 과거 어느 시점에서든 그래프의 상태를 조회할 수 있습니다.

## 개요

시간 여행 시스템은 모든 프로퍼티 변경에 대해 불변의 버전 체인을 유지합니다. 프로퍼티를 업데이트하면 이전 값이 타임스탬프와 함께 버전 저장소에 보존됩니다. 이를 통해 다음이 가능합니다:

- `AT TIME`을 사용한 **시점 쿼리**
- `BETWEEN TIME`을 사용한 **범위 쿼리**
- 생성/삭제 타임스탬프를 포함한 **시간 여행 엣지 버전 관리**
- 버전화된 데이터에 대한 **시간 여행 집계**

<Callout type="info">
시간 여행 기능은 `temporal-core` 기능 플래그가 필요합니다 (기본 활성화). 시간 여행 엣지 속성을 사용하려면 `temporal-edge`도 활성화하세요.
</Callout>

## AT TIME 쿼리

특정 시점의 그래프 상태를 조회합니다:

```cypher
MATCH (p:Person {name: 'Alice'})
AT TIME '2024-01-01T00:00:00Z'
RETURN p.name, p.age
```

이후에 어떤 변경이 이루어졌든, 2024년 1월 1일 시점의 Alice 프로퍼티를 반환합니다.

### 동작 원리

1. 쿼리 엔진이 대상 타임스탬프를 식별합니다
2. 매칭된 각 노드/엣지에 대해 버전 체인을 순회합니다
3. 요청된 시점에 활성화된 버전을 찾습니다
4. 해당 버전에서 프로퍼티를 해석합니다

## BETWEEN TIME 쿼리

시간 범위에 걸친 프로퍼티 변경을 조회합니다:

```cypher
MATCH (p:Person {name: 'Alice'})
BETWEEN TIME '2024-01-01T00:00:00Z' AND '2024-12-31T23:59:59Z'
RETURN p.name, p.age, p._version_timestamp
```

2024년 동안 활성화된 Alice 프로퍼티의 모든 버전을 반환합니다.

## 시간 여행 엣지 버전 관리

<Callout type="warning">
시간 여행 엣지 버전 관리는 `temporal-edge` 기능 플래그가 필요합니다.
</Callout>

엣지도 시간 여행 메타데이터를 가지고 있습니다. 관계가 생성되거나 삭제된 시점을 조회할 수 있습니다:

```cypher
MATCH (a:Person)-[r:KNOWS]->(b:Person)
WHERE r._created_at > '2024-06-01T00:00:00Z'
RETURN a.name, b.name, r._created_at
```

### 엣지 시간 여행 프로퍼티

| 프로퍼티 | 설명 |
|----------|-------------|
| `r._created_at` | 관계가 생성된 타임스탬프 |
| `r._deleted_at` | 관계가 삭제된 타임스탬프 (활성 상태이면 null) |

## 시간 여행 집계

시간 여행 쿼리와 집계 함수를 결합할 수 있습니다:

```cypher
MATCH (p:Person)
BETWEEN TIME '2024-01-01' AND '2024-12-31'
RETURN count(p) AS person_count, avg(p.age) AS avg_age
```

## 버전 저장소 아키텍처

버전 저장소는 프로퍼티별 불변 체인으로 구현됩니다:

```
현재 값 --> 버전 N --> 버전 N-1 --> ... --> 버전 1
          (타임스탬프)  (타임스탬프)         (타임스탬프)
```

- 각 버전은 프로퍼티 값과 변경 타임스탬프를 기록합니다
- 체인은 추가 전용입니다 (이전 버전은 수정되지 않음)
- 스토리지 오버헤드는 전체 데이터 크기가 아닌 변경 횟수에 비례합니다

## 모범 사례

<Steps>

### ISO 8601 형식의 타임스탬프 사용

CypherLite는 ISO 8601 형식의 타임스탬프를 파싱합니다. 모호성을 피하기 위해 항상 타임존 정보를 포함하세요.

### 필요한 시간 여행 기능만 활성화

`temporal-core` (기본)는 노드 수준 버전 관리를 제공합니다. 엣지 생성/삭제 타임스탬프가 필요한 경우에만 `temporal-edge`를 활성화하세요. 엣지당 스토리지 오버헤드가 추가됩니다.

### 스토리지 증가 고려

모든 프로퍼티 변경은 새로운 버전 항목을 생성합니다. 쓰기 집중 워크로드의 경우 데이터베이스 파일 크기를 모니터링하고 필요시 주기적 압축을 계획하세요.

</Steps>
