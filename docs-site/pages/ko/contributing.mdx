# 기여 가이드

CypherLite에 기여해 주셔서 감사합니다. 이 페이지는 기여 가이드라인의 요약을 제공합니다. 전체 가이드는 GitHub의 [CONTRIBUTING.md](https://github.com/Epsilondelta-ai/CypherLite/blob/main/CONTRIBUTING.md)를 참조하세요.

## 버그 리포트

[GitHub Issues](https://github.com/Epsilondelta-ai/CypherLite/issues)를 사용하여 버그를 보고하세요. 다음을 포함해 주세요:

- 문제에 대한 명확한 설명
- 재현 단계
- 예상 동작 vs 실제 동작
- Rust 버전 (`rustc --version`) 및 OS

## 변경 사항 제출

<Steps>

### 저장소 Fork

[CypherLite](https://github.com/Epsilondelta-ai/CypherLite)를 Fork하고 `main`에서 브랜치를 생성하세요.

### 브랜치 이름 지정

설명적인 브랜치 이름을 사용하세요: `feat/my-feature` 또는 `fix/issue-123`.

### 변경 작업

아래의 코드 스타일과 테스트 가이드라인을 따르세요.

### Pull Request 생성

`main`에 대한 PR을 열고 변경 사항에 대한 명확한 설명을 첨부하세요.

</Steps>

<Callout type="info">
중요한 변경의 경우 구현에 시간을 투자하기 전에 먼저 이슈를 생성하여 접근 방법을 논의하세요.
</Callout>

## 개발 환경 설정

**요구사항:**

- Rust 1.84 이상 (MSRV)
- [rustup](https://rustup.rs/)을 통한 표준 Rust 툴체인

**설정:**

```bash
git clone https://github.com/Epsilondelta-ai/CypherLite.git
cd CypherLite
cargo build --workspace
cargo test --workspace --all-features
```

## 기능 플래그

| 플래그 | 설명 |
|------|-------------|
| `temporal-core` | 시간 여행 그래프 지원 (기본) |
| `temporal-edge` | 시간 여행 엣지 속성 |
| `subgraph` | 서브그래프 추출 |
| `hypergraph` | 하이퍼그래프 지원 (`subgraph` 필요) |
| `full-temporal` | 모든 시간 여행 기능 |
| `plugin` | 플러그인 시스템 |

## 코드 스타일

- 커밋 전 `cargo fmt` 실행
- `cargo clippy --all-features` 실행 및 모든 경고 수정
- 모든 새 기능에 대한 테스트 작성
- 패키지당 최소 85% 테스트 커버리지 유지
- `cargo test --workspace --all-features`로 검증

## 커밋 메시지

[Conventional Commits](https://www.conventionalcommits.org/) 준수:

- `feat(core): add temporal aggregation support`
- `fix(storage): resolve WAL replay edge case`
- `docs: update architecture diagram`
- `test(query): add parser edge case tests`
