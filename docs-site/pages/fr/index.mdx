## Pourquoi CypherLite ?

<div className="feature-grid">

<div className="feature-card">
### Zero Configuration
Ouvrez un fichier et commencez a interroger. Pas de processus serveur, pas d'installation, pas de fichiers de configuration. Juste un fichier `.cyl`.
</div>

<div className="feature-card">
### Base de Donnees Mono-fichier
Toutes les donnees du graphe sont dans un seul fichier portable. Copiez, sauvegardez ou partagez votre base de donnees entiere en une seule operation.
</div>

<div className="feature-card">
### Conforme ACID
Securite transactionnelle complete avec Write-Ahead Logging, isolation par instantane et recuperation apres crash. Vos donnees sont toujours coherentes.
</div>

<div className="feature-card">
### Bibliotheque Embarquee
Integrez CypherLite directement dans votre application. Disponible comme crate Rust, package Python, module Go, addon Node.js et bibliotheque C.
</div>

<div className="feature-card">
### Systeme de Plugins Extensible
Quatre types de plugins (ScalarFunction, IndexPlugin, Serializer, Trigger) vous permettent d'etendre CypherLite selon vos besoins.
</div>

<div className="feature-card">
### Requetes Temporelles
Interrogez votre graphe a n'importe quel moment dans le temps. Le versionnage integre suit chaque modification des noeuds, aretes et proprietes.
</div>

</div>

## Exemple Rapide

```rust
use cypherlite_query::CypherLite;

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let db = CypherLite::open("my_graph.cyl")?;

    db.execute("CREATE (a:Person {name: 'Alice', age: 30})")?;
    db.execute("CREATE (b:Person {name: 'Bob', age: 25})")?;
    db.execute(
        "MATCH (a:Person {name: 'Alice'}), (b:Person {name: 'Bob'}) \
         CREATE (a)-[:KNOWS {since: 2023}]->(b)"
    )?;

    let result = db.execute(
        "MATCH (p:Person) WHERE p.age > 20 RETURN p.name, p.age"
    )?;
    for row in result {
        let row = row?;
        println!("{}: {}", row.get("p.name").unwrap(), row.get("p.age").unwrap());
    }
    Ok(())
}
```

## Fonctionnalites Principales

| Fonctionnalite | Description |
|---------|-------------|
| **Property Graph** | Nodes and edges with typed properties (Null, Bool, Int64, Float64, String, Bytes, Array) |
| **Cypher Queries** | openCypher subset with MATCH, CREATE, MERGE, SET, DELETE, RETURN, WHERE |
| **Temporal Versioning** | AT TIME queries, version store, temporal edge versioning |
| **Subgraphs** | Named subgraphs as first-class entities with CREATE/MATCH SNAPSHOT |
| **Hyperedges** | N:M relationships connecting arbitrary numbers of nodes |
| **Plugin System** | ScalarFunction, IndexPlugin, Serializer, Trigger plugin types |
| **FFI Bindings** | C ABI, Python (PyO3), Go (CGo), Node.js (napi-rs) |
